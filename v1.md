# AgentDesk — V1 Product Idea

> **One-line pitch:** A pixel office where you watch your AI coding agents work in real time, paired with a mission control panel where you and your agents actually collaborate — no more hunting through terminal logs.

---

## The Problem

Developers running multiple AI coding agents (Claude Code, Codex CLI, Cursor, etc.) face a brutal UX problem:

- You open 3–5 terminal tabs, each running an agent on a different task
- Every tab is a wall of machine-readable logs
- You have no idea what each agent is doing without manually switching tabs
- Agents silently stop and wait for your input — you find out 30 minutes later
- You lose track of what's been done, what's running, and what's queued
- Cross-machine visibility is zero — if you work on laptop + desktop, you're blind

**The current "solution" is tab-switching and hope.** Nobody has built the control layer that should sit on top of these agents.

---

## The Solution — AgentDesk

AgentDesk is two things combined into one product:

### 1. 🎮 The Pixel Office (Observation Layer)
A real-time visual office where each of your agents becomes an animated pixel character. You see, at a glance:

- Which agents are actively working (typing animation)
- Which are reading/searching files (reading animation)
- Which are waiting for your input (alert bubble)
- Which have finished their task (idle at desk)
- What tool they just used — shown as a floating indicator above their head

No terminal-switching. One beautiful view tells you everything.

### 2. 🛰️ Mission Control (Collaboration Layer)
A structured dashboard where you and your agents work together, not just observe. This is not just logs — it's a real workspace:

- **Task Board** — Kanban-style: Queue → In Progress → Done. See every task across all agents
- **Live Activity Feed** — What each agent did, in plain English, timestamped
- **Agent Chat** — Send instructions to a specific agent without leaving the dashboard
- **Cost & Token Tracker** — Real-time spend per agent, per session, per project
- **Stuck Agent Alerts** — Notification (Slack, email, in-app) when an agent needs your input
- **Session History** — Full replay of what an agent did in any past session
- **Cross-device Sync** — Open the dashboard on your phone, see what's running on your desktop

## How It Works (Technical V1)

```
Your Machine                          AgentDesk Cloud
─────────────────                     ──────────────────────
Claude Code / Codex                   
    ↓ writes JSONL logs               
AgentDesk Hook (tiny npm package)  →  Secure WebSocket Stream
    ↓ watches log files               ↓
    ↓ reads Claude hooks              Dashboard (Web App)
                                      ├── Pixel Office view
                                      ├── Mission Control panel
                                      ├── Task board
                                      └── Cost tracker
```

**Setup is one command:**
```bash
npm install -g agentdesk
agentdesk connect
```

That's it. No config, no code changes. AgentDesk reads existing Claude Code JSONL transcripts and hook events silently — purely observational. Nothing in your agent's behavior changes.

## Who Is This For

**Primary user:** Solo developer or indie hacker running 2–5 Claude Code or Codex sessions simultaneously, building projects fast, and losing time managing terminal chaos.

**Secondary user:** Small dev team (2–5 people) where multiple developers each run agents and want shared visibility into what's happening across the team.

**Not yet for:** Large enterprises, non-developer users, teams running cloud-hosted agents (different problem).

## What Makes This Defensible

- **First mover advantage** — being the first polished SaaS in this space matters enormously for developer tools
- **Network effects** — team plans create shared dashboards, pulling in more users
- **Data moat** — aggregate (anonymized) agent behavior data becomes valuable for product improvements and benchmarking
- **Switching cost** — once developers customize their office layout, set up alerts, and build workflow habits around AgentDesk, they don't leave
- **Breadth of agent support** — each new tool added (Cursor, Gemini CLI, etc.) expands the addressable market
